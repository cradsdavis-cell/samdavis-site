Why this approach — the Karpathy wiki pattern

Most people use LLMs like chat: ask, get answer, history fades. The LLM has to rediscover context every time. You repeat yourself.

The wiki pattern is different. Instead of throwing prompts at the LLM, you incrementally build a persistent knowledge base — a structured, interlinked collection of markdown files. The LLM doesn't just retrieve and answer; it reads, integrates new info into the existing wiki, updates cross-references, flags contradictions. Knowledge compounds.

Three layers:

  1. Raw sources (references/) — articles, files, notes you drop in. Immutable. The LLM reads but doesn't modify these.
  2. The wiki (notes/) — what the LLM builds + maintains: summary pages, entity pages, concepts, connections. The LLM owns this layer entirely.
  3. The schema (CLAUDE.md) — tells the LLM how the wiki is structured + conventions to follow. Makes the LLM a disciplined wiki-maintainer rather than a generic chatbot.
