## Top 3 priority fixes (impact-ranked)

### 1. BLOCKING — Unify the `/book/*` flow with `lib/site.css` register

**User impact:** The highest-conversion path on the site (the entire booking flow) reads as a different brand than the rest. A founder arriving from `/offer → /book/coaching-block` sees the boutique register evaporate at the moment they're being asked for $500.

**Fix:**
- Delete `--ink: #1f1d1a;`, `--paper: #fafaee;`, `--accent-soft: rgba(143, 117, 48, 0.08);` from `book/_styles.css:3-7`. They duplicate (and slightly miss) the existing `--ink-deep`, `--bg-main`, `--accent-glow` tokens.
- Replace `background: #fff` (lines 53, 79, 82, 85) → `background: var(--bg-soft)` for cards and policy boxes. Pure white doesn't exist anywhere else on the site and reads as Stripe-default.
- Replace `color: #fff` on `.book-cta` (line 66) and `.slot-button.selected` (lines 95-96) → `color: var(--bg-main)` — matches the `cta-primary` pattern in `lib/site.css:157`.
- Replace `.book-cta:hover { background: #6B5624; }` (line 77) → `var(--accent-deep)`.
- Replace `.slot-error { color: #b00020; padding: ...; background: #fee; }` (line 115) → introduce `--err: #8B2A1F` and `--err-bg: #F4E4DE` tokens in `lib/site.css` palette, then reference. Pure red on pink doesn't sit in the cream/brown/sage palette.
- Apply `h1` styling from `lib/site.css:69-73` to `.book-header h1` — currently `font-size: 2.4rem` with no `font-family` (inherits sans, not Playfair). This is the single biggest brand-break on the site.

### 2. BLOCKING — Booking flow nav is a different component

**User impact:** Booking pages use a fixed-floating pill nav (`position: fixed; top: 10px; ... border-radius: 999px`) while the marketing layer uses a full-width header bar with a brand mark + logo image. Same site, two navs. The pill nav also has no brand mark / logo / wordmark — it's anonymous.

**Fix:** Replace the pill-nav definition in `book/_styles.css:21-45` with the canonical `.site-nav-bar` from `lib/site.css:193-244` (full-width header, brand image + wordmark, nav-links group). The pill pattern is fine for *print* pages (`/onepager`, `/build`, `/build-onepager`) where it's an overlay on an A4 document — keep it scoped there. Booking pages are screen-first interactive pages and should match the marketing nav.

Same issue exists on `/onepager` (line 341-368), `/build` (line 401-426), `/build-onepager` — these duplicate the pill nav definition four times. Even if you keep the pill pattern for print pages, move the definition into `lib/site.css` as a `.site-nav-pill` utility, not inline-redefined per page.

### 3. FLAG — Mobile hero on landing crops to a "floating-head" composition

**User impact:** Below 900px, `index.html:42-50` flips the hero to single-column with `.hero-illustration { order: -1; max-width: 240px; margin: 0 auto; }`. The result is a 240×240 cropped portrait floating alone above the eyebrow text. On a phone, the illustration is the first thing visible and dominates the viewport without any text context.

**Fix:** Either (a) reduce mobile illustration to `max-width: 160px` so the eyebrow + H1 enter the viewport on first paint, or (b) inline the illustration *beside* the H1 at small sizes via a 2-column grid that holds 100px illustration + flex-grow text — same visual rhythm as the desktop hero, just compressed. The current order:-1 placement is what most sites do, but it's the wrong call for a site whose hero copy is the conversion engine.

## Detailed findings

### Pillar 1 — Typographic hierarchy (7/10)

**Pass — strong baseline.** `lib/site.css:60-119` defines a clean ramp: Playfair 800 for h1-h4 (with italic em-emphasis in accent brass), Inter for body (18px / line-height 1.65), mono for eyebrows + meta. `clamp(40px, 6vw, 72px)` for h1 is exactly the boutique pattern used by stripe.press + sequoiacap. The italic em-morph (`lib/site.css:75-79`) for accent words inside headings is a signature move that ties the brand together — anthropic.com uses the same trick.

**FLAG — `book/_styles.css:49` overrides H1.** `.book-header h1 { font-size: 2.4rem; margin-bottom: 0.5rem; color: var(--accent); }` — no `font-family`, so this H1 falls back to whatever `body` sets, which is Inter (sans). The boutique-serif identity vanishes. Worse: making H1 the accent brass color instead of `--ink-deep` breaks the brass-as-accent rule the rest of the site obeys. Same issue propagates to every `/book/*/index.html` since they all use this CSS.

**FLAG — `.book-header p { font-size: 1.1rem; color: var(--ink-soft); }`** (line 50) — this is doing lede duty but without the `.lede` system (`clamp(18px, 1.6vw, 23px); line-height: 1.5`). Use `.lede` class directly; the system already exists.

**FLAG — `/onepager`, `/build`, `/build-onepager` define their own H1 sizes** (`onepager/index.html:70-77` sets `font-size: 36px`; `build/index.html:68-75` sets `clamp(38px, 5.2vw, 66px)`). These are print/slide pages so per-page sizing makes some sense, but the tokens aren't named — every page reinvents the H1 ramp. Consider a `--h1-print` / `--h1-slide` token in `lib/site.css` for cross-page consistency.

**FLAG — `.book-card h2 { font-size: 1.35rem; color: var(--accent); }`** (`_styles.css:60`) — h2 sized smaller than the surrounding p (h2 = 1.35rem ≈ 21.6px, body = 18px) and coloured accent. The h2 has lost its hierarchical weight; on the `/book` grid the card titles read like meta labels rather than offer names. Bump to `font-size: clamp(20px, 2vw, 24px); color: var(--ink-deep); font-family: var(--serif);`.

**PASS — eyebrow + meta usage is consistent** across `/`, `/overview`, `/offer`, `/onepager`. Mono 12px / 0.22em letter-spacing in accent brass is unmistakable site signature.

### Pillar 2 — Spacing + rhythm (7/10)

**Pass — generous boutique cadence on marketing pages.** `lib/site.css:30-36` defines a scale `--space-1: 8px → --space-7: 200px` and the section padding `var(--space-6) 0` (120px) on `lib/site.css:144` is the boutique tell. The hero on `/` runs `padding-top: var(--space-7)` (200px) which is exactly stripe.press / anthropic.com territory.

**FLAG — `/book/*` collapses to `padding: 5rem 2rem 3rem;`** (`_styles.css:48`). That's 80px top / 32px sides / 48px bottom — about *half* the vertical rhythm of the marketing pages. The booking flow feels cramped relative to the page it just came from.

**FLAG — `.book-card { padding: 2rem 1.5rem; }`** + `.book-policy { padding: 1.5rem; }` use rem-based spacing while the rest of the site uses the named scale tokens. The mismatch is intentional (the `/book/*` flow is its own thing) but it shouldn't be — convert to `var(--space-4) var(--space-3)` and the rhythm aligns automatically.

**FLAG — `.work-entry { padding: var(--space-3) 0; border-bottom: 1px solid var(--rule); }`** on `index.html:55-58` — 24px vertical inside each entry is tight given the body copy is 100+ words. Try `--space-4` (40px) to let each Carbon Tracker / Derwen / This system entry breathe. Currently they merge visually.

**FLAG — `.rung` padding on `/offer`** (`offer/index.html:21-24`): `padding: var(--space-4) 0;` between rungs but the rungs themselves have no internal horizontal padding because they're inside `.read-col` (640px max). Reads as one long scrolling column without container — the rungs need a sense of being *contained* via slight inset background (`background: var(--bg-soft); padding: var(--space-4); border-radius: 6px;`) or a leading vertical accent bar.

**PASS — section dividers via `.section + .section { border-top: 1px solid var(--rule); }`** (`lib/site.css:147-149`) is a quietly elegant pattern — no visible rule unless sections stack, no rule below the last section. Stripe.press uses the same approach.

### Pillar 3 — Colour palette discipline (4/10)

This is the lowest-scoring pillar and where the highest-leverage fix sits.

**BLOCKING — `book/_styles.css:3-7` redefines tokens that already exist.**
```css
:root {
  --ink: #1f1d1a;           /* duplicates and slightly misses --ink-deep #3E3418 */
  --paper: #fafaee;         /* duplicates and slightly misses --bg-main #FAF7EE */
  --accent-soft: rgba(143, 117, 48, 0.08);   /* close to --accent-glow but different opacity */
}
```
These aren't called by anything inside `_styles.css` (I checked — the file uses `var(--ink)` and `var(--paper)` for `body` declaration only, and `--accent-soft` is defined-but-unused). Delete the block; cascade from `lib/site.css`.

**BLOCKING — `book/_styles.css` uses `#fff` six times** (lines 53, 66, 79, 82, 85, 95, 96, 101). Pure white doesn't exist anywhere else in the palette. The cream `--bg-main: #FAF7EE` is the site's "white" — and the cards should sit on `--bg-soft: #F2EEDD` for a subtle elevation. Currently the cards are pure-white islands floating on cream, which is the visual signature of a Stripe checkout, not a boutique consulting site.

**BLOCKING — `book/_styles.css:77` hardcodes `#6B5624` for the CTA hover.** That value *is* `--accent-deep` from `lib/site.css:18`. Hardcoded hexes that match named tokens are worse than divergent ones, because they silently de-couple when the token gets updated. Replace with `var(--accent-deep)`.

**FLAG — `book/_styles.css:115` `.slot-error { color: #b00020; background: #fee; }`** — these are Material Design's red palette dropped into a warm-earth context. The error pill looks like it teleported from a different site. Add `--err: #8B2A1F` (a desaturated red that sits inside the warmth) + `--err-bg: #F4E4DE` (a peach-tinted cream) to the palette in `lib/site.css`, reference here.

**FLAG — `.book-card-hero::before { background: var(--accent); color: #fff; }`** — the "★ Most popular" badge is brand-aligned in background but still uses pure white text. Change to `color: var(--bg-main)` for cohesion.

**PASS — marketing pages do use the tokens.** `/`, `/overview`, `/offer` reference `var(--accent)`, `var(--ink-deep)`, `var(--bg-soft)`, `var(--rule)` throughout. The discipline is real; it just stops at the `/book` directory boundary.

**PASS — print/slide pages use the tokens correctly.** `/onepager`, `/build-onepager`, `/build` all reference the named tokens. The only deviation is the pill-nav backdrop `rgba(250, 247, 238, 0.93)` (the hex of `--bg-main` at 93% alpha) — this is genuinely necessary for the backdrop-blur translucency, but it should still be expressed as `color-mix(in srgb, var(--bg-main) 93%, transparent)` for token cohesion.

### Pillar 4 — Composition + alignment (6/10)

**FLAG — Landing hero mobile collapse** (see priority fix #3 above). The 240×240 illustration above the eyebrow on a 375px mobile viewport occupies ~50% of the first paint with no copy context. Recommend dropping to `max-width: 160px` or inlining beside the H1.

**FLAG — `/overview` + `/offer` heroes are 200px-grid with 140px illustration** (`overview/index.html:13-20`, `offer/index.html:12-20`). The 200px column with 140px image inside it creates a 60px gap on the right that the text doesn't fill (the text starts at the next grid column). Two options: (a) reduce grid to `120px 1fr` so the illustration grid column matches the illustration size, or (b) anchor the image right with `justify-self: end` so the empty 60px sits between img and text deliberately.

**FLAG — Booking grid card with `★ Most popular` badge has the badge overflowing the card border** (`_styles.css:100-101`): `position: absolute; top: -10px; left: 16px;`. On mobile-narrow viewports where the card width approaches 280px (`grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));`), the badge sits on top of the card border but there's no offset for the page edge. On the smallest mobile viewport the badge can clip to the left page edge. Add `left: clamp(16px, 4vw, 24px);` or shift to `top: 8px; left: 50%; transform: translateX(-50%);` for a centered tab badge.

**FLAG — Footer `.location` (`lib/site.css:274-280`) is italic 13.5px sans** but the rest of the footer is monospace 14px. The italic shift adds a nice typographic note but the smaller size + sans-shift makes "Coogee, Sydney" feel like an afterthought rather than a deliberate close. Bump to `font-size: 14px` and keep italic.

**FLAG — `/book` page H1 + lede are left-aligned but cards are auto-fit grid centered**. The header sits left-justified while the card grid below it can have 1-4 cards depending on viewport. On a 1200px viewport with 4 cards laid out, the header is left and the cards stretch full width — fine. On a 700px viewport with 2 cards, the cards are 2-column and the header is still left — feels off-balance. Either left-align everything via `max-width: var(--wrap-max); margin: 0 auto;` on the container (currently `max-width: 1080px`), or center-align everything for a boutique register.

**PASS — `.read-col { max-width: 640px; }`** holds reading copy to a single human-readable measure. Every essay-style section on `/overview` and `/offer` obeys it. This is anthropic.com's signature move and the site has it right.

**PASS — Footer alignment** (`lib/site.css:247-280`) — wordmark + email + linkedin + phone + location stacked left-aligned with `line-height: 2` is a tasteful close.

### Pillar 5 — Motion + interaction polish (8/10)

**PASS — `prefers-reduced-motion` honored.** `lib/site.css:350-358` zeroes out float-gentle, italic-morph transitions, and scroll-reveal transitions/transforms. This is the correct accessibility-first pattern.

**PASS — `cta-primary` hover has subtle arrow-translate** (`lib/site.css:166-175`): `::after` arrow translates 6px on hover with a 0.2s ease. Small, considered, brand-aligned. Anthropic uses the same pattern.

**PASS — Italic morph is gentle.** 4s interval, 0.6s crossfade, 6px translateY entrance/exit. Doesn't scream for attention. The phrase rotation on the H1 (`chief of staff would → longtime colleague would → thoughtful editor would`) is the single most memorable detail on the homepage.

**PASS — Scroll reveal is restrained.** 0.6s opacity + 12px translateY entry, threshold 0.15, rootMargin -60px. Stagger via `--reveal-index * 80ms` is delicate (most sites overshoot with 150-200ms). The 80ms feels boutique.

**FLAG — Float-gentle on hero illustration** is set to `4s ease-in-out infinite` (4px translateY). Subtle, but combined with the italic-morph cycling at the same time on the H1, there are two competing motion loops in the same hero. On reduced-motion it's fine; on default it's slightly fidgety. Consider a longer period (8s) for the float or no motion at all on the illustration.

**FLAG — Slot picker buttons have hover but no transition.** `_styles.css:84-93` defines `.slot-button:hover { border-color: var(--accent); }` with no `transition:` property. The border color change is instant — feels uncommitted. Add `transition: border-color 0.15s ease, background 0.15s ease;`.

**FLAG — `.pat:hover { transform: translateX(4px); }`** on `/build` slide 3 (`build/index.html:139`) is great motion but has no `transition`. The hover is instant. Add `transition: transform 0.2s ease;` to the `.pat` rule.

**PASS — Smooth-scroll on anchor clicks** (`lib/site.js:66-79`) updates URL hash without a jump. Tiny detail but the right one.

### Pillar 6 — Brand consistency (5/10)

**The two-site problem.** Marketing layer (`/`, `/overview`, `/offer`) is sequoia/anthropic-tier: cream paper, brass accents, Playfair italic emphasis, monospace eyebrows, generous whitespace. Booking layer (`/book/*`) is Stripe-checkout-tier: white cards, sans-serif H1s, rem-based spacing, pure-white CTAs, pure-red error states. A user who arrives via `/offer → /book/coaching-block` experiences a brand transition at the moment of conversion. This is the most expensive UX issue on the site.

**FLAG — Nav inconsistency.** Three nav patterns coexist:
- Marketing pages: full-width `.site-nav-bar` with brand mark, image, wordmark, current-page indicator.
- Booking pages (5 files): same `.site-nav-bar` class but no brand image, no wordmark, just text links separated by `·`.
- Print pages (3 files): floating pill nav, fixed top center, semi-transparent with backdrop blur.

Either the pill nav becomes the universal pattern (and the marketing pages adopt it) or the marketing nav becomes universal (and print pages keep the pill as overlay-on-A4). The current state — 3 different navs across 11 visible pages — is the strongest possible signal of "stitched together."

**FLAG — Favicon path mismatch.** Marketing pages reference `/lib/img/favicon.png` + `/lib/img/favicon.ico`. Booking pages reference `/favicon.svg` *and* `/lib/img/favicon.png`. Pick one.

**FLAG — Title casing inconsistency.** Site uses sentence case for H1/H2 (`How we work together`, `An AI executive assistant that chief of staff would`) — boutique-correct. But CTA button text is title case (`Book a 30-minute call`, `Book + Pay →`) and book card H2s are also title case (`Coaching Block`, `EA Basic Build`). The mix is fine for CTAs and proper nouns; just noting that the *titles* on `/book/coaching-block.html` etc are `Coaching Block` (title case) while the marketing register prefers sentence case.

**FLAG — `/book` nav says "For founders"** but the link goes to `/offer`. Marketing pages call it "Offer" in the nav. Pick one. ("For founders" reads warmer; "Offer" reads tighter. For the boutique register, "Offer" wins because it doesn't presume the user identity.)

**PASS — Footer is identical across all marketing pages.** Wordmark + email + LinkedIn + phone + location. The location italic `Coogee, Sydney.` close is a quietly memorable signature.

**PASS — H1 + italic accent pattern** is unmistakable as the brand. Once you've seen one page, you can recognise others by silhouette alone.
